Actividad 23 de Abril


-   Listar los requisitos de acuerdo a a idea que seleccionaron ayer como más apropiada.
-   Priorizar los requisitos con los criterios:
•   Importancia para el usuario.
•   Complejidad de Implementación.

Verificar que los requisitos tengan coherencia y que listen todo lo necesario para la
solución.

Salida esperada: Lista de requisitos (los de mayor prioridad) que incluirán en sus
prototipos conceptuales.

Quienes ya tienen más claro sus requisitos, podemos ir trabajando el diseño de la solución.

Coherencia de Requisitos


- El software tiene todos los datos necesarios para producir la salida esperada.
- El usuario (u otra parte del software) consume la información producida por cada parte
  del software.
- El software ofrece navegabilidad a cada funcionalidad del software.
